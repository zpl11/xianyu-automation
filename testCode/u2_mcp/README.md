# 闲鱼 Android 自动化执行库 (MCP + uiautomator2)

本目录（`d:\code\闲鱼\testCode\u2_mcp`）是我们在对抗闲鱼动态风控过程中，沉淀下来的**最核心、最稳定的自动化脚本库**。
之前所有碎片化的、基于盲目坐标点击的测试脚本均已被清理，目前保留的都是经过“实战动态验证”的健壮逻辑。

## 🎯 核心架构与原理
我们的系统目前由两大部分组成：
1. **AI 控制中枢**：利用 MCP 协议（`server.mjs`）连接大模型，下发 Python 指令。
2. **底层雷达引擎**：利用 `uiautomator2` 通过 ADB (`192.168.1.58:5555`) 实时获取闲鱼 App 的 XML DOM 树进行状态机验证（遇弹窗点弹窗，遇拦截走备用逻辑）。

## 📂 核心文件清单

### 1. 业务执行脚本 (Task Scripts)
这些是你今后日常自动化最常用的“武器”，每个脚本内部都内置了大量的异常处理和节点校验。

*   **`task_publish_service.py` (原发布脚本)**
    *   **功能**：全自动上架“服务定制类”虚拟商品。
    *   **核心攻坚点**：能够自动从相册选择首图，并**完美绕过了闲鱼针对副业/服务的“预计工期”拦截弹窗**。同时通过故意触发风控漏洞，**实现了极难配置的【价格私聊】标签**，是引流矩阵的核心。
*   **`task_search_workbuddy.py` (原搜索抓取脚本)**
    *   **功能**：在首页安全激活搜索框，强制清空残留的热搜缓存，精准搜索任意关键词并滚动抓取竞品数据（标题、价格、销量）。
    *   **核心攻坚点**：解决了闲鱼自定义输入框“无法获取焦点、残留输入”的隐形 Bug。
*   **`task_scrape_feed.py` (原瀑布流抓取脚本)**
    *   **功能**：在任意信息流页面进行无限滚动拉取，实现去重统计。

### 2. 底层基建与工具 (Infrastructure)
*   **`debug_state.py`**：当你发现自动化卡住时，单独运行此脚本，它会瞬间把平板当前屏幕的所有节点打在终端上，让你拥有“透视眼”。
*   **`u2_bridge.py`**：连接 ADB 和视觉模型（PaddleOCR）的底层桥接层。
*   **`server.mjs` / `package.json`**：Node.js 写的 MCP Server 服务端，用于打通 LLM 和本地 Python 执行环境。

## 💡 今后的自动化使用规范 (Best Practices)
1. **绝不盲打**：任何长链路操作，必须拆分成“操作 -> 确认页面 -> 再操作”的状态机模式。
2. **警惕自定义组件**：遇到输入失效，不要强行 `send_keys`，要先模拟物理点击（`click`）激活焦点区域。
3. **拥抱异常**：把闲鱼的风控弹窗（如“你还没有填写价格”）当成业务节点。我们甚至可以利用这些异常拦截机制，卡出闲鱼原本不轻易开放的功能。

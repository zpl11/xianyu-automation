# UI 自动化控制框架拓扑

## 架构概述
本代码仓库实现了一个双层自动化架构，通过 `uiautomator2` 桥接与 Android 界面进行交互。该设计模式严格将无状态的交互基元（原子层）与复杂、依赖状态的业务执行流（场景层）分离开来。

### 1. 原子控制域 (`atomic_tools/`)
基础核心层，负责执行与 Android UI 拓扑层级和硬件输入的离散式、非上下文交互。

**核心组件：** `xianyu_atomic_tools.py`
- **设备实例化：** 单例模式的设备连接流。
- **UI 节点遍历：** 提取完整的界面布局 XML。
- **确定性寻址：** 基于精确/模糊文本匹配及内容描述进行精准的节点调用。
- **空间交互：** 绝对坐标调用机制与物理长按仿真。
- **输入流注入：** 高速 I/O 数据流覆写及内存缓冲区替换（剪贴板接管）。
- **运动学仿真引擎：**
  - *贝塞尔滚动*：生成多项式参数曲线（三次贝塞尔曲线），用于执行非线性、随机化坐标的滑动轨迹。
  - *安全区滚动*：调用底层 Android `InputManager`（通过 shell 通道），映射到特定的屏幕向量（左侧边界安全区），以防止引发非预期的 UI 状态突变。

### 2. 复合状态机域 (`business_scenarios/`)
上层集成流，通过引入原子级基元构建条件执行图及自动化状态转移矩阵。

**业务执行管道：**
- `auto_publish_product.py`：高韧性的 13 节点流转状态机。
  - 实现了前置依赖检测（网络健康度验证）、备选 UI 遍历回退、基于剪贴板的内存负载注入、启发式分类匹配（正则引擎），以及递归式自动异常捕获与处理（如：在阻塞拦截状态下动态感知并补全缺失属性）。
- `click_avatar.py`：空间及层级节点解析协议。
  - 运用边界框包含逻辑和几何距离测算矩阵，将松散的 XML 元素强相关，实现在复杂覆盖层拓扑下的精准元素定位（解决遮挡点击失效）。
- `check_wifi_connection.py`：自动化网络接口环境自检与自愈子系统。
- `get_search_suggestions.py`：用于动态特征数据提取的自动化 I/O 序列。

## 运行依赖
- Python 3.x 核心环境
- `uiautomator2` 桥接库
- 目标 Android 终端设备（需保持 ADB 调试通信管道激活）。

## 综合执行流水线
此仓库拓扑强制执行了严格的关注点分离策略。原子模块被降维至标准的硬件/OS I/O 接口级，完成对底层 API 的抽象隔离。场景模块剥离了控制层，专注于条件逻辑处理与结构化推演，支持对界面交互进行独立态隔离调试，切断了与顶层应用状态的隐性耦合。

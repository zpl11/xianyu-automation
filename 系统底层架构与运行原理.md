# 闲鱼全自动操控矩阵：底层架构与运行原理

本文档用于解析当前系统是如何实现“让 AI 自主操作 Android 平板并完成闲鱼闭环任务”的技术链路。

---

## 🧱 一、 系统的四层异构网络架构

整个系统打通了一条从“云端自然语言”到“物理屏幕点击”的精密指令管道：

1. **第一层：云端战略大脑（LLM 大语言模型）**
   * **角色定位**：指挥官与程序员。
   * **运行机制**：AI 接收人类的自然语言指令（如“去搜竞品”），理解业务逻辑并直接生成/修改底层的 Python 执行代码。AI 是这套系统的控制核心。

2. **第二层：中枢神经网关（MCP 协议层）**
   * **角色定位**：跨次元通信桥梁。
   * **运行机制**：运行在本地宿主机的 Node.js MCP (Model Context Protocol) 服务（`server.mjs`）。它赋予了云端 AI **实体交互权**。有了它，AI 才能在本地写入 Python 文件，并拉起系统终端去执行命令。

3. **第三层：肌肉驱动引擎（Python + `uiautomator2`）**
   * **角色定位**：物理动作执行者。
   * **运行机制**：利用开源自动化测试框架 `uiautomator2`。它负责将抽象的业务代码转化为 Android 底层的系统调用（如：计算坐标、注入滑动矢量、模拟屏幕点按）。

4. **第四层：末端硬件受控体（Android 平板 + ADB 调试桥）**
   * **角色定位**：主战场（闲鱼 App）。
   * **运行机制**：平板通过局域网（`192.168.1.58:5555`）开放 ADB 端口，并常驻运行 `atx-agent` 守护进程，负责实时接收来自第三层的物理注入指令并回传屏幕 XML 状态。

---

## 🔄 二、 AI 自主运行飞轮（感知-决策-行动）

AI 之所以能“看懂并操作”闲鱼，全靠代码中植入的**闭环状态机**机制，彻底摆脱了传统的“盲点宏脚本”。

### Step 1: 感知（雷达扫描）
每次动作执行前，脚本必定调用 `d.dump_hierarchy()`。
瞬间提取平板当前屏幕的完整 XML DOM 树。无论是常规按钮还是风控弹窗，都会被转化为带有绝对坐标的文本数据传回内存。这就是 AI 的“眼睛”。

### Step 2: 决策（风控绕过与目标锁定）
通过解析回传的 XML，脚本进行极速判断：
*   **异常识别**：如果发现 `text="预计工期"`，判定为触发了 NLP 风控，立即进入备用分支处理必填项。
*   **目标匹配**：如果发现 `text="领红包"`，计算该节点的 bounds 坐标 `[x1, y1][x2, y2]`，提取中心点。

### Step 3: 行动（物理注入）
锁定坐标后，通过 ADB 向 Android 发送硬件级中断信号（如 `click` 或 `send_keys`）。特别针对闲鱼的防劫持组件，强制采用“物理坐标点击 + 多次 DEL 清空”的策略来完成输入。

### Step 4: 监控自愈（AI 兜底介入）
在脚本静默运行时，AI 在云端通过 MCP 实时监听本地的运行日志（Log 终端流）。
如果脚本因遇到未知 UI 崩溃退出，AI 会立刻下发 `debug_state.py`，透视当前死锁界面的隐形节点，并光速重构执行脚本，实现**全自动的断点自愈**。
